How to use grove
1

Detect sprints

Scan any repo for sprint directories by finding claims.json files and reading git history.

$ npx @grainulator/grove detect-sprints
2

Generate manifest

Build a wheat-manifest.json that maps topics to claims and files. One file for the entire sprint state.

$ npx @grainulator/grove generate-manifest
Real example

Running grove on a repo with 3 sprints

A mono-repo with multiple research sprints. Grove finds them all, shows their status, and generates a manifest.

$ grove detect-sprints --json { "sprints": [ { "path": "sprints/graphql-migration", "claims": 14, "status": "active", "last_commit": "2h ago" }, { "path": "sprints/auth-scaling", "claims": 23, "status": "active", "last_commit": "1d ago" }, { "path": "sprints/perf-baseline", "claims": 41, "status": "done", "last_commit": "5d ago" } ], "active_sprint": "sprints/graphql-migration" } $ grove generate-manifest --root sprints/graphql-migration Manifest written to wheat-manifest.json (4.2KB) Topics: 6 Claims: 14 Files: 8 AI sessions can now read wheat-manifest.json instead of scanning the full repo.
Key features

Shared infrastructure for research

Tools that every grainulator package uses. Also work standalone.

Sprint Detection

Finds sprint directories by scanning for claims.json. Uses git history to rank and pick the active sprint.

Manifest Generation

Builds a compact JSON index of topics, claims, and files. New AI sessions read this instead of scanning everything.

HTML Templates

Dark scroll-snap explainers, dashboard layouts, and presentation templates used by wheat and mill.

PDF Builder

Convert markdown to PDF via npx md-to-pdf. No local install needed -- heavy lifting on demand.

Zero Dependencies

Node built-in modules only. No npm install surprises, no supply chain concerns.

AI-Friendly Output

Manifest stays under 10KB. Works with Claude Code's Glob, Grep, and Read tools out of the box.
